On coding

很是怀念高中当时使用pascal的感觉，遇到问题心中便能有相应的代码开始构建，从架构到具体细节，十分清晰，进入大学后丢掉了pascal，新学的语言由于练习较少熟练度不高，也很难找到当初那样的感觉。

用python写exp是老早就有的想法，之前虽然也写过一些扫描、盲注、批量利用的小脚本，但是相比于稍成熟的工程还是有很大的差距，这里通过对前辈工程学习为契机，总结一下python编写的一些基础知识。

getopt

getopt模块是专门处理命令行参数的模块，用于获取命令行选项和参数，也就是sys.argv。命令行选项使得程序的参数更加灵活。支持短选项模式-和长选项模式--。

该模块提供了两个function和一个exception:

getopt.getopt()

getopt.getopt(args, options[,long_options]):
解析命令行选项和参数列表。

Arguments

args：是对于当前进程没有引导的参数列表；sys.argv[1:]
options：是脚本所需要的选项字母的字符串，若options后面加有:则该选项要跟参数；
long_options:是程序所需要的长选项名字列表。在命令行中，程序针对用户的输入进行前缀匹配，例如，long_options集合为['foo','frob'],选项 --fo将被匹配为 --foo,但是--f不会被匹配，因此提出GetoptError().长选项名后的等号(=)表示如果设置该选项，必须有附加的参数，否则就不附加参数。

Return Value

返回值包含两个参数

  • 一个是由(option ,value)构成的list
  • 另一个是被分割好的的程序参数list

options顺序为被发现的顺序，会出现长短混合的情况。

getopt.gnu_getopt()

getopt.gnu_getopt(args, options[,long_options]):
GUN风格的getopt()功能与前者类似，这意味这选项和参数会混杂。

exception getopt.GetoptError

当未定义的选项出现，或者有需要参数的选项没有给参数时触发该例外。该例外的参数是表明错误原因的一个字符串。对于长参数来说，一个选项若不需要参数，但被传参，同样触发例外。
